Introduction

Welcome

Welcome to my container based capture the flag (CTF). I hope you have fun and maybe learn a few things along the way.

Containers are a very popular emerging technology in the industry right now. They offer many benefits during the product lifecycle from development to production and more. Although you may not be familiar with containers don't worry! One of my goals is to show you that the process of testing the security of a container based infrastructure is similar to that of traditional web systems with which we have experimented.

CTF Name

Every technology project needs a "cool" name, usually something greek or mythology related. Khôra was a term used by the ancient Greek philospher Plato to describe a type of seperated space. Containers are also a sort of seperated space.

CTF Structure

This capture the flag guides you through the process of a complete customer engagement with a fake company. You will be refered to as "ACME Pen Testing" and the company is named "ACME Company". To start you will be given a fake letter of engagement from the company. From this you will find flags throughout this company's infrastructure. Flags will be clearly visible right after you gain access to a system. They will all mention "This is CTF flag x!".

If you want to be a purist the letter of engagement should be enough to help you solve the entire puzzle. However each flag will be accompanied by a guide on this wiki website. With varrying levels of hints available, including the complete solutions. Resources under the /ctf/ directory on this server are specifically for helping facilitate the CTF, don't scan them or look too much into anything in this directory. All the flags are elsewhere.

Technical Details

This capture the flag is meant to be run using Docker Compose. It can be accessed from any machine which can talk to the host machine running Docker Compose. A Docker container named workbench is provided pre-setup with Kali Linux + Metasploit + all the tools you need to complete this CTF. Open a bash console into this container by running docker-compose exec workbench bash, see tools for more.

Docker Compose creates a subnet for the containers in this CTF. The ID of this Docker Compose subnet changes based on your computer. Inside the workbench container it is easy to determine the subnet ID, simply run ip addr show eth0. This subnet will be accessible from inside and outside the containers. On your host machine traffic can be sent to this subnet via loopback 127.0.0.1. This CTF will refer to resources as being on 127.0.0.1, however you should replace this subnet ID with whatever subnet your containers are running on.
